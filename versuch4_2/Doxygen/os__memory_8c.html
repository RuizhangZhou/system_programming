Detailed Description

Contains functions to allocate blocks of any spos-memory-drivers. We manage the blocks by creating a lookup table with one nibble per byte payload. So we have 1/3 map and 2/3 use-heap of each driver.

Important: Always use addresses of the use-heap and convert them into the nibbles of the map (only) if needed. For this purpose, there are helper functions such as os_getMapEntry(...).
