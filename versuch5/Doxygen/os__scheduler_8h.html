◆ os_enterCriticalSection()

void os_enterCriticalSection ( void  )

Enters a critical code section by disabling the scheduler if needed. This function stores the nesting depth of critical sections of the current process (e.g. if a function with a critical section is called from another critical section) to ensure correct behaviour when leaving the section. This function supports up to 255 nested critical sections.

◆ os_exec()

ProcessID os_exec ( ProgramID  programID,
Priority  priority 
)

This funtion is used to execute a program that has been introduced with os_registerProgram. A stack will be provided if the process limit has not yet been reached. themselves, simulating TinyOS 2 scheduling (just kick off interrupts ;) ).

Parameters
programIDThe program id of the program to start (index of os_programs).
priorityA priority ranging 0..255 for the new process:
  • 0 means least favourable
  • 255 means most favourable Note that the priority may be ignored by certain scheduling strategies.
Returns
The index of the new process or INVALID_PROCESS as specified in defines.h on failure

◆ os_registerProgram()

ProgramID os_registerProgram ( Program *  program)

Used to register a function as program. On success the program is written to the first free slot within the os_programs array (if the program is not yet registered) and the index is returned. On failure, INVALID_PROGRAM is returned. Note, that this function is not used to register the idle program.

Parameters
programThe function you want to register.
Returns
The index of the newly registered program.
